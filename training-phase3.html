モジュール07: レセプト点検 ★★★★★

🔍
なぜこのモジュールが最重要なのか

医療機関のレセプト業務は、受付・診療・入力・会計を経て「レセプト作成」で完了するように見える。しかし作成されたレセプトをそのまま審査支払機関に提出することは極めて危険である。

レセプトには数十〜数百の算定項目が含まれ、各項目に個別の算定要件・回数制限・適応症・コメント要件が存在する。これらを日々の入力で完全に把握することは現実的に不可能であり、レセプト点検はエラーを請求前に発見・修正する「最終防衛ライン」である。

BPO事業者にとって、レセプト点検は最もクライアントへの付加価値を発揮できる業務である。算定漏れの発見による収益増加、返戻・査定率の低下、医師への算定教育フィードバック — BPO費用を超えるROIを証明しやすい。

1. このモジュールで学ぶこと

本モジュールを修了すると、以下を説明・実行できるようになる。

  1. レセプト点検の全体像 — 点検の定義・目的・種類・業務フロー上の位置づけを体系的に説明できる
  2. 点検4種の実施 — 内容点検・縦覧点検・突合点検・資格点検を目的と手順を区別して実施できる
  3. 点検チェックリストの活用 — 基本情報・傷病名・診療行為・投薬・検査の各カテゴリを網羅的にチェックできる
  4. 算定ルール違反パターンの識別 — 代表的な返戻・査定事例15件を挙げ、発生原因と対策を説明できる
  5. 3段階点検フローの運用 — 1次（システム）・2次（目視）・3次（管理者）の各フローを設計・運用できる
  6. BPO受託時の品質設計 — クライアント向けの点検体制・SLA・品質報告の枠組みを設計できる
Level 3目標: このモジュールは「指導可能レベル」が到達目標。自分が実行できるだけでなく、クライアントのスタッフに点検手順を教え、品質を評価できるレベルを目指す。

2. 点検を怠った場合のリスク

リスク発生メカニズム医療機関への影響
返戻審査機関が記載不備・要件不充足と判断 → 請求を差し戻し翌月以降の再請求が必要。キャッシュフロー悪化
査定審査機関が過剰算定と判断 → 一部または全額を削除して支払い算定分が回収できず実質的な収益減
監査不正請求が疑われる場合の行政調査最悪の場合は保険医療機関の指定取消し

4. 点検の種類

4.1 内容点検（コンテンツチェック）

レセプトの診療内容が診療報酬点数表のルールに適合しているかを確認する。最も基本的かつ作業量の多い点検。

確認カテゴリ具体的な確認内容主なエラー例
算定ルール違反同時算定禁止項目の重複、算定回数超過初診料と再診料の同時算定、週2回制限超過
記載漏れ必須コメント・症状詳記の欠如特定薬剤のコメント未記載
コメント不備コメントの内容が不十分・様式コード誤り「症状詳記」の内容が薄い
傷病名と診療行為の不整合傷病名に対応しない診療行為の算定高血圧症の傷病名で感染症治療薬を処方
点数誤算定点数マスタ更新漏れによる旧点数での算定改定後の点数が反映されていない

4.2 縦覧点検（ロンギチュードチェック）

同一患者の複数月にわたるレセプトを横断的に確認する点検。「縦覧」とは「縦（時間軸）に一覧する」意味。

縦覧点検が必要な主な理由:

  1. 算定回数制限の跨月確認 — 「前○ヶ月以内」「年1回」など月跨ぎの制限が多数
  2. 転帰の整合性 — 「治癒」記載の翌月に同じ傷病名が再登場していないか
  3. 慢性疾患の傷病名管理 — 継続疾患が途中で消えていないか
  4. 初診・再診の区分誤り — 前月まで通院があるのに今月が初診料算定になっていないか
算定制限確認対象期間
月2回制限当月のみ確認
月1回制限当月＋前月
3ヶ月制限当月＋前2ヶ月
6ヶ月制限当月＋前5ヶ月
年1回制限当月＋前11ヶ月
実務ポイント: 支払基金は「縦覧チェック」として算定回数・傷病名の経時的整合性を自動チェックしており、医療機関側での事前確認が必須。

4.3 突合点検（マッチングチェック）

医科レセプトと調剤レセプトを突合（照合）し、整合性を確認する点検。

確認項目医科側調剤側確認内容
処方薬と調剤薬の一致処方箋内容調剤内容処方した薬と調剤された薬が一致しているか
傷病名と処方薬の適応傷病名調剤薬傷病名に対応した薬が処方されているか
日数制限処方日数調剤日数投薬日数制限（向精神薬等）への適合
用量の適切性処方量調剤量規定用量内か（過剰投与の疑い）
2024年改定ポイント
突合点検の強化が明記された。特に向精神薬・オピオイド系薬剤については審査機関の突合チェックが厳格化されており、処方側の傷病名管理が一層重要。

4.4 資格点検（エリジビリティチェック）

患者の保険資格が診療月において有効であったかを確認する点検。

手段概要確認タイミング
オンライン資格確認（マイナ保険証）マイナポータルに紐づいた保険情報をリアルタイム確認受付時（2024年以降原則義務化）
被保険者証の目視確認有効期限・記号番号の確認受付時
レセプト返戻後確認「被保険者資格なし」返戻を受けた後の調査返戻後（事後対応）
電話照会保険者への電話での資格確認疑義が生じた場合

#チェック項目確認内容注意点

7.3 1次点検（レセコン自動チェック）

目的: システムが検知可能な機械的エラーを全件一括で排除する。

チェック機能検知できるエラー例限界
マスタ照合薬剤コード・診療行為コードの誤りコードが正しくても内容が誤りの場合は検知不可
算定回数チェック月内回数制限超過跨月の回数制限は月次単体では検知できない
必須コメントチェックコメントコード未入力コメントの内容の適切性は判断不可
保険情報チェック法別番号の形式誤り実際の資格有効性は確認不可
金額計算チェック点数・金額の計算誤り点数自体が正しいかは判断不可
BPO実務の視点
1次点検でのエラー件数・エラー率を月次でKPIとして記録すること。エラー率が上昇した月は入力担当の教育強化のサインである。

7.4 2次点検（スタッフ目視チェック）

モジュール08: 医師確認・承認

👩‍⚕️
なぜ「点検後」に医師確認が必要なのか

Module 07で点検技術を習得したが、BPO事業者が「おかしい」と気づいても独断で修正することは絶対にできない。レセプトの診断・治療内容は医師の医学的判断に基づくものであり、最終的な答えを出せるのは診察した医師だけである。

BPO事業者の点検は、医師が見落としやすい事務的・制度的なエラーを発見し、「これを医師に確認してほしい」という橋渡し役を担うプロセスである。この工程を効率よく運用できるかどうかが、BPOサービスの価値の核心を決める。

1. このモジュールで学ぶこと

  1. 医師確認が必要なケースを即座に判断できる — 疑義・病名・コメント等について、医師確認が必要かBPO側で解決できるかを区別し、適切に仕分けられる
  2. 疑義照会リストを体系的に作成できる — 医師への質問を項目別・優先度別に整理し、一度の確認で漏れなく回答を得られる照会リストを作成できる
  3. 症状詳記の必要条件を判断し、依頼文を作成できる — リスクの高いレセプトに対し、適切な症状詳記の添付を医師に依頼できる
  4. 承認プロセスを管理できる — 医師の回答を記録・反映し、承認後のルールに従って適切にレセプトを最終化できる
  5. クライアント医師との協働関係を設計できる — 医師の負担を最小化する照会フォーマット・タイミング・コミュニケーションルールを設計できる

3. 医師確認・承認の全体像

3.1 法的根拠

医師による診療報酬請求への関与は、保険医療機関及び保険医療養担当規則（療担規則）によって義務づけられている。

療担規則 第8条: 保険医は、患者の診療を行った場合には、遅滞なく、診療録に当該診療に関し必要な事項を記載しなければならない。

療担規則 第22条: 保険医療機関は、療養の給付に関する費用の請求を行わなければならない。

つまり、保険医が診療内容を正確に記録する義務を負い、その記録に基づいて保険医療機関として請求が行われる。BPO事業者はあくまでも代行・補助であり、内容の正確性についての最終責任は常に医師・医療機関に帰属する。

3.2 医師とBPO事業者の権限範囲

区分実施主体内容
医師が行うこと保険医診断・治療行為の実施、病名の確定・転帰の決定、症状詳記の記載、最終承認
BPO事業者が行うことBPO担当者診療録とレセプトの突合・整合性確認、算定ルールの照合、疑義事項の整理・照会、回答のレセプトへの反映
BPO事業者が行ってはならないこと—病名の独断追加・削除、診療内容の無断修正、症状詳記の代筆（捏造）、承認を省略した請求
BPO担当者が「明らかに不要な算定」と判断した場合でも、医師の確認なしに項目を削除してはならない。疑義として医師に提示し、判断を仰ぐことが正しい手順である。

4. 確認が必要なケース

同月に同種の処置が算定条件を超えて算定診療録を確認し、医師に実施の有無・回数を確認
病名のない処置・検査が算定対応病名を追加するか、算定を取り消すか
入院・外来の区分違いによる算定算定根拠の確認
加算要件を満たさない加算が算定要件充足の確認（施設基準届出、文書交付等）

算定漏れの疑い:

ケース確認事項
診療録に実施記録があるが算定されていない算定追加の可否を確認
当然行うべき管理料が未算定算定漏れか意図的な非算定かを確認
在宅医療で関連加算が漏れている実施記録と照合し、追加算定の可否を確認

4.3 コメント記載の要否判断

算定項目必要なコメント
時間外緊急院内検査加算検査実施の時刻
夜間・休日等加算診察開始時刻
長期投薬（28日超）長期投薬が必要な理由
特定薬剤治療管理料薬剤名・測定値
在宅自己注射指導管理料指導管理の対象薬剤名

4.4 症状詳記が必要な高額レセプト

ケース目安
入院レセプトで高額1件当たり100万円超が一般的な基準
長期に渡る入院90日を超える入院（特に精神科・療養病棟）
高価薬の使用分子標的薬・抗がん剤等で薬剤費が高額
多剤投与1患者への投薬種類が多い場合
過去に査定を受けた診療行為の再算定前回査定の経緯の説明が必要
BPO事業者の役割
BPO担当者は症状詳記の「必要性の判断」と「医師への依頼」を行う。症状詳記の文章そのものは医師が記載する（代筆は不可）。ただし、記載すべき観点や構成の提案は行ってよい。

5.1 照会リストの基本構成

項目内容

診療年月対象月
照会カテゴリ病名 / 算定 / コメント / 症状詳記 / その他

作成のポイント: 医師が選択式で回答できるよう「□ 病名追加する / □ 算定削除する」など選択肢を用意する。専門用語より日常業務の言葉で記載し、医師が短時間で把握できるようにする。

5.2 照会タイミングの設計（まとめ照会）

タイミング内容
翌月5日前後点検完了後、照会リストをまとめて送付

回答期限前日未回答分のリマインダー連絡

5.3 照会の手段

手段特徴適した状況
書面（照会リスト）記録が残る、非同期で対応可能標準的なケース
電子メール＋添付記録が残る、迅速テレワーク環境下・遠方の医師
対面（診療時間外の打合せ）即時回答、ニュアンスの共有が可能複雑な案件・初回オンボーディング
電話迅速だが記録が残りにくい緊急時のみ（後でメール等で確認）

5.4 回答の記録と反映

  1. 回答内容の記録: 照会リストに医師の回答を正確に転記。日付・回答者名を必ず記録
  2. レセプトへの反映: 回答に基づきレセプトを修正（病名追加・算定取消・コメント追加等）
  3. 反映後の再確認: 修正後のレセプトを再点検し、意図通りに反映されているか確認
  4. 未回答事項の管理: 期限までに回答が得られなかった項目は、対応方針を医師に最終確認する

6. 症状詳記

6.1 審査側が見るポイント

審査側が見るポイント医師が記載すべき内容

長期化の理由入院・投薬が長期に及んだ医学的な根拠
薬剤選択の根拠高価薬を使用した場合、その薬剤でなければならない医学的理由

上記患者のレセプトについて、審査機関への症状詳記の添付をお願いいたします。

添付が必要な理由:

記載いただきたいポイント:

  • 当該治療を選択した医学的根拠

7. 承認プロセス

7.1 最終承認のタイミング

タイミング内容
翌月9日前後レセプト最終版を医師に提出し、内容確認・承認を依頼
翌月10日（提出期限当日）承認後、審査機関への提出
原則: 医師の最終承認なしに審査機関へ提出することは厳禁。電子レセプトでは医師の電子署名が記録されることが一般的。

7.2 承認記録の管理

項目内容
対象医療機関名クライアントの医療機関
対象診療年月承認を受けた診療月
承認日時医師が承認した日時
承認方法書面押印 / 電子署名 / システム承認操作等
承認者名承認した医師の氏名
承認対象件数その月の承認レセプト件数

記録の保管期間は5年以上が望ましい（療担規則で診療録は3年保存義務があるが、紛争対応・監査対応の観点から5年以上を推奨）。

7.3 承認後の修正ルール

原則として、医師の承認後にBPO事業者がレセプト内容を変更することは禁止。

承認後に変更が必要になった場合の手順:

  1. 変更の必要性と内容を明確にし、医師に説明する
  2. 医師の確認・再承認を得る
  3. 変更内容と再承認の記録を残す
  4. 変更後のレセプトを正式版として管理する
例外: 明らかな入力ミス（患者氏名の誤字等の事務的エラー）は、医師への報告の上、軽微なものは確認のみで修正が認められる場合もある。ただし、算定内容・病名・点数に関わる変更は必ず再承認を要する。

8. BPO視点 — 医師との協働設計

8.1 契約・業務開始時に取り決めるべき事項

取り決め事項具体例
照会の手段と窓口「Gメールで疑義照会リストを送付→医師が直接返信」
照会の頻度・タイミング「毎月5日前後に一括送付、8日までに回答依頼」
回答のフォーマット「選択肢に○を付けて返信」
緊急時の連絡方法「提出期限当日の緊急時は電話で直接確認」
症状詳記の依頼期限「毎月7日までに提出」
最終承認の方法「システム上の承認ボタン」または「紙の承認書に押印」
複数医師の場合の分担「各担当患者の疑義は担当医師へ、全体は院長が最終承認」

8.2 照会の効率化

手法内容
まとめ照会月1回、すべての疑義を一括で照会。都度の連絡を排除
選択式フォーマット「□ 病名追加する / □ 削除する」のように○×をつけるだけの形式
優先度の明示「高優先度3件、低優先度8件」と明示し、対応順序を判断しやすくする
前月との差分表示毎月同じ患者が登場する場合は「前月と変更なし」と明記
患者情報の集約患者ID・氏名・疑義内容を1行にまとめ、スクロールなしで概要把握

8.3 医師の負担を最小化する工夫

工夫具体的な実装
背景情報の提示疑義が生じた理由（算定ルール上の問題点）を簡潔に添付
推奨案の提示「BPO担当者の判断では病名追加が適切と思われます」と案を添える（最終判断は医師）
Q&A形式の整理繰り返し発生するパターンをQ&A化し、定型処理で対応可のルールを事前合意
月次レポートの提供疑義件数・修正内容サマリーを医師にフィードバックし、経営改善につなげる
